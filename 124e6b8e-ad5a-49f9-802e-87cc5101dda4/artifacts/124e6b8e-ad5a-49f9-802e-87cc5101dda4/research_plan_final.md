# Understanding the Mediation Role of Mobility Patterns in the Relationship between Socioeconomic Status and Depression in South Carolina Counties

## Research Question
How do spatial patterns of mobility (place visitation) mediate the relationship between neighborhood social economic status and depression prevalence across the counties of South Carolina?

## Objectives

1. **Estimate the county-level association between socioeconomic status and depression prevalence across South Carolina**
   - Hypothesis: Counties with lower socioeconomic status (lower Med_income and percent_>=HighSch; higher percent_Poverty, percent_Unemployed, and percent_Uninsured) have higher Depression_prev.
   - Steps:
     a. Coerce GEOID to string in SC_ACS_data and SC_PLACES_depression; retain only GEOID, Med_income, percent_Poverty, percent_Unemployed, percent_Uninsured, percent_>=HighSch, percent_Black, percent_Hispanic, percent_>=18, and Depression_prev; drop rows with missing GEOID and report counts of missing values in Depression_prev and each ACS field.
     b. Left-join Depression_prev from SC_PLACES_depression to SC_ACS_data on GEOID, keeping GEOID, Depression_prev, Med_income, percent_Poverty, percent_Unemployed, percent_Uninsured, percent_>=HighSch, percent_Black, percent_Hispanic, and percent_>=18.
     c. Compute SES_index as the mean of z-scored Med_income and percent_>=HighSch and the negated z-scores of percent_Poverty, percent_Unemployed, and percent_Uninsured; append SES_index to the table.
     d. Fit OLS model Depression_prev ~ SES_index + percent_>=18 + percent_Black + percent_Hispanic and compute HC1 robust standard errors; save coefficient estimates, robust standard errors, t-statistics, p-values, R-squared, and residual diagnostics.
     e. Join county_ses_depression_with_index.csv to SC_counties on GEOID to create a polygon layer with GEOID, Depression_prev, SES_index, Med_income, percent_Poverty, percent_Unemployed, percent_Uninsured, percent_>=HighSch, percent_Black, percent_Hispanic, and percent_>=18.

2. **Quantify how county place-visitation composition varies with socioeconomic status across South Carolina**
   - Hypothesis: Counties with lower Med_income and percent_>=HighSch and higher percent_Poverty, percent_Unemployed, and percent_Uninsured have systematically different visitation composition (category shares) and lower mobility_diversity (Shannon entropy) across POI categories.
   - Steps:
     a. Left-join SC_Visitor_POI to county_ses_depression_with_index.csv on GEOID, retaining GEOID, Depression_prev, Med_income, percent_Poverty, percent_Unemployed, percent_Uninsured, percent_>=HighSch, percent_>=18, percent_Black, percent_Hispanic, and visitation count columns Full_Service_Restaurant, Sport_facilities, Parks, Fastfood_Restaurant, Convenience, Supermarket, Warehouse, Fruit, TobaccoStore, and DrinkingPlaces.
     b. Compute total_visits as the row-wise sum of Full_Service_Restaurant + Sport_facilities + Parks + Fastfood_Restaurant + Convenience + Supermarket + Warehouse + Fruit + TobaccoStore + DrinkingPlaces; for counties where total_visits > 0 compute share_Full_Service_Restaurant, share_Sport_facilities, share_Parks, share_Fastfood_Restaurant, share_Convenience, share_Supermarket, share_Warehouse, share_Fruit, share_TobaccoStore, and share_DrinkingPlaces as category/total_visits; set all share_* to 0 where total_visits == 0.
     c. Compute mobility_diversity as Shannon entropy across the 10 share_* columns (share_Full_Service_Restaurant, share_Sport_facilities, share_Parks, share_Fastfood_Restaurant, share_Convenience, share_Supermarket, share_Warehouse, share_Fruit, share_TobaccoStore, share_DrinkingPlaces) using the convention 0 * log(0) = 0; append mobility_diversity.
     d. Standardize the 10 share_* variables to zero mean and unit variance and run PCA; append mobility_PC1 (first principal component score) and mobility_PC2 (second principal component score) to each county.
     e. Fit OLS models (with HC1 robust standard errors): mobility_PC1 ~ Med_income + percent_Poverty + percent_Unemployed + percent_Uninsured + percent_>=HighSch + percent_>=18 + percent_Black + percent_Hispanic and mobility_diversity ~ Med_income + percent_Poverty + percent_Unemployed + percent_Uninsured + percent_>=HighSch + percent_>=18 + percent_Black + percent_Hispanic; save coefficient estimates, robust standard errors, t-statistics, p-values, and R-squared for both models.

3. **Test whether county mobility composition mediates the socioeconomic status–depression association and evaluate spatial dependence**
   - Hypothesis: mobility_PC1 and/or mobility_diversity transmit indirect effects from Med_income, percent_Poverty, percent_Unemployed, percent_Uninsured, and/or percent_>=HighSch to Depression_prev, and accounting for spatial dependence changes inference on socioeconomic and mobility effects.
   - Steps:
     a. Fit the mediator model mobility_PC1 ~ Med_income + percent_Poverty + percent_Unemployed + percent_Uninsured + percent_>=HighSch + percent_>=18 + percent_Black + percent_Hispanic using OLS with HC1 robust standard errors; save coefficients and fitted values per GEOID.
     b. Fit the outcome model Depression_prev ~ Med_income + percent_Poverty + percent_Unemployed + percent_Uninsured + percent_>=HighSch + mobility_PC1 + percent_>=18 + percent_Black + percent_Hispanic using OLS with HC1 robust standard errors; save coefficients and residuals per GEOID.
     c. Estimate mediation of each socioeconomic variable (Med_income, percent_Poverty, percent_Unemployed, percent_Uninsured, percent_>=HighSch) → Depression_prev through mobility_PC1 using nonparametric bootstrap resampling of counties: for each bootstrap sample, refit the mediator model mobility_PC1 ~ Med_income + percent_Poverty + percent_Unemployed + percent_Uninsured + percent_>=HighSch + percent_>=18 + percent_Black + percent_Hispanic and the outcome model Depression_prev ~ Med_income + percent_Poverty + percent_Unemployed + percent_Uninsured + percent_>=HighSch + mobility_PC1 + percent_>=18 + percent_Black + percent_Hispanic; compute ACME, ADE, and total effect for each socioeconomic variable (with the remaining socioeconomic variables included as covariates) with percentile bootstrap confidence intervals.
     d. Create a Queen contiguity adjacency/weights table from SC_counties polygons keyed by GEOID, storing each GEOID–neighbor_GEOID pair and an associated weight (row-standardized).
     e. Subset and reorder sc_county_queen_weights.csv to include only GEOIDs present in outcome_model_results.csv and to match the GEOID ordering used for the residual vector; output the aligned weights table.
     f. Compute global Moran’s I for the Depression_prev model residuals using the aligned Queen contiguity weights; report Moran’s I, expectation, variance, z-score, and permutation p-value.
     g. Fit a spatial error model for Depression_prev with predictors Med_income, percent_Poverty, percent_Unemployed, percent_Uninsured, percent_>=HighSch, mobility_PC1, percent_>=18, percent_Black, and percent_Hispanic using the aligned Queen contiguity weights; save regression coefficients, the spatial error parameter (lambda), standard errors, and model fit statistics.
     h. Fit a geographically weighted regression (GWR) for Depression_prev with predictors Med_income, percent_Poverty, percent_Unemployed, percent_Uninsured, percent_>=HighSch, mobility_PC1, percent_>=18, percent_Black, and percent_Hispanic using county centroids derived from SC_counties (joined by GEOID); select bandwidth via cross-validation or AICc; save local coefficient estimates (including intercept) per GEOID, bandwidth choice, model fit diagnostics, and summaries of spatial variability in coefficients.
